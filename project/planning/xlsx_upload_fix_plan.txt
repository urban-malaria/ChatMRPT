# Fix XLSX File Upload Support

## Problem
XLSX files upload but don't work because:
1. All files saved as `raw_data.csv` regardless of original format (an Excel file becomes `raw_data.csv` but still contains Excel binary)
2. Code uses `pd.read_csv()` which fails on Excel content

## Solution
Fix both sides:
1. **Saving**: Preserve correct extension (`raw_data.csv` vs `raw_data.xlsx`)
2. **Reading**: Create helper to find and read the data file with correct reader

---

### Step 2: Create helper function (and reuse encoding logic)

**File: `app/data_analysis_v3/core/encoding_handler.py`**

Add helpers that encapsulate both discovery *and* proper reading:
```python
from pathlib import Path


def find_raw_data_file(session_folder: str) -> str | None:
    """Find the raw data file in a session folder (csv or xlsx)."""
    for ext in ['csv', 'xlsx', 'xls']:
        path = Path(session_folder) / f'raw_data.{ext}'
        if path.exists():
            return str(path)
    return None


def read_raw_data(session_folder: str, **kwargs) -> pd.DataFrame:
    """Find and read the raw data file from a session folder."""
    filepath = find_raw_data_file(session_folder)
    if not filepath:
        raise FileNotFoundError(f"No raw data file in {session_folder}")

    ext = Path(filepath).suffix.lower()
    if ext in ('.xlsx', '.xls'):
        return pd.read_excel(filepath, **kwargs)
    else:
        # Preserve existing encoding fixes for CSVs
        return EncodingHandler.read_csv_with_encoding(filepath, **kwargs)
```
