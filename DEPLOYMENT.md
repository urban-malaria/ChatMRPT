# ChatMRPT Deployment Guide for Render

## Prerequisites

1. Render account with a free or paid plan
2. OpenAI API key for ChatGPT functionality
3. Settlement data files (436MB) - stored separately due to size

## Deployment Steps

### 1. Initial Setup on Render

1. Fork or connect this repository to your Render account
2. Create a new Web Service
3. Connect to your GitHub repository
4. Render will automatically detect the `render.yaml` configuration

### 2. Environment Variables

Set the following environment variables in Render dashboard:

- `OPENAI_API_KEY`: Your OpenAI API key (required)
- `FLASK_ENV`: `production` (automatically set by render.yaml)
- `SECRET_KEY`: Will be auto-generated by Render

### 3. Settlement Data Setup

Due to the large size of settlement data (436MB), it's not included in the Git repository. After deployment:

1. Access your Render service shell
2. Navigate to `/app/kano_settlement_data/Kano_clustered_footprint/`
3. Upload the following files:
   - `kano_grids_clustered.dbf` (321MB)
   - `kano_grids_clustered.shp` (109MB)
   - `kano_grids_clustered.shx` (6MB)
   - `kano_grids_clustered.prj` (145B)

**Option A: Using Render Shell**
```bash
# In Render shell
cd /app/kano_settlement_data/Kano_clustered_footprint/
# Use wget or curl to download from your storage
wget https://your-storage-url/kano_grids_clustered.dbf
wget https://your-storage-url/kano_grids_clustered.shp
wget https://your-storage-url/kano_grids_clustered.shx
wget https://your-storage-url/kano_grids_clustered.prj
```

**Option B: Using rclone (if available)**
```bash
# Configure rclone for your cloud storage
rclone copy remote:path/to/settlement_data ./kano_settlement_data/
```

### 4. Disk Storage

The `render.yaml` configures 10GB of persistent disk storage mounted at `/app/instance`. This stores:
- Uploaded user data files
- Generated reports
- Application logs
- SQLite database

### 5. Health Check

The application includes a health check endpoint at `/ping` that Render monitors.

## Post-Deployment Verification

1. Access your deployed URL: `https://your-app-name.onrender.com`
2. Test the following:
   - File upload functionality
   - Chat interface responsiveness
   - Settlement map generation
   - Report generation

## Troubleshooting

### Common Issues

1. **GDAL Installation Errors**
   - The `apt-packages` file includes all necessary geospatial dependencies
   - Ensure GDAL version matches requirements.txt (3.6.2)

2. **Settlement Maps Not Working**
   - Verify settlement data files are properly uploaded
   - Check file permissions: `chmod -R 755 kano_settlement_data/`

3. **Memory Issues**
   - Consider upgrading from free tier if processing large datasets
   - Implement data sampling for visualization (already configured)

### Monitoring

- Check Render logs for any startup issues
- Monitor disk usage to ensure sufficient space
- Review application logs in `/app/instance/logs/`

## Performance Optimization

1. **Caching**: The application implements visualization caching
2. **Sampling**: Large datasets are intelligently sampled for performance
3. **Service Container**: Singleton services reduce memory overhead

## Security Notes

- Never commit `OPENAI_API_KEY` to the repository
- Keep settlement data access restricted
- Regular security updates via `pip install --upgrade`

## Updates and Maintenance

1. **Automatic Deploys**: Enabled in render.yaml for main branch
2. **Manual Updates**: Push to main branch or use Render dashboard
3. **Database Migrations**: Not required (using SQLite with auto-schema)
