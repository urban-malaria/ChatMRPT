================================================================================
KEY MIGRATION DECISIONS
================================================================================

1. USER JOURNEY DRIVEN:
   Path from signup → upload → TPR/analysis → visualization → planning
   → Each stage has its own directory

2. ONE-BRAIN AGENT (app/agent/):
   - Core orchestration in request_interpreter.py
   - Execution engines (executor.py, executor_simple.py)
   - Tool management (base.py, complete_analysis_tools.py)
   - State management (state.py, state_manager.py)

3. TPR WORKFLOW (app/tpr/):
   - Isolated intent classification
   - Language interface for natural language → TPR mapping
   - Analysis tools specific to TPR
   - Separated from general agent concerns

4. VISUALIZATION (app/visualization/):
   - All rendering: maps, charts, explanations
   - Map and settlement tools
   - Visual explanation service

5. SHARED INFRASTRUCTURE (app/services/):
   - LLM manager and adapters
   - Session/memory management
   - Data loading and repositories
   - Analysis helpers and formatters

6. ALL ROUTES → app/api/:
   - Clean separation of Flask routes from logic
   - Sub-routes in app/api/analysis/
   - All route files have _routes.py suffix (or routes.py)

   Every moved file needs import updates. This is the largest effort.
   Examples:

     from app.tools.base import ... → from app.agent.tools.base import ...
     from app.web.routes.core_routes import ... → from app.api.core_routes import ...

2. FILE RENAMES:
   ~40 files get renamed when moved (see "RENAME" column in CSV).
   This requires updating both import statements AND module references.

3. NEW __init__.py FILES:
   Create __init__.py in each new directory if it doesn't exist.

4. TEST COVERAGE:
   After migration, run:
     python -m pytest tests/ -v
   To catch any import/routing issues.

   Routes in app/api/ need to be registered in the Flask factory.
