CONTEXT: WHY THIS MATTERS
-------------------------
This is part of the "Smart TPR Combination Switching" feature. The goal is to
allow users to switch between different TPR combinations (facility level + age
group) without re-uploading data or recalculating from scratch.

There are 16 possible TPR combinations:
- 4 facility levels: primary, secondary, tertiary, all
- 4 age groups: u5 (under 5), o5 (over 5), pw (pregnant women), all_ages

When a user completes the TPR workflow selecting ONE combination, we want to
pre-compute the OTHER 15 combinations in the background so they can quickly
switch later by saying "Switch to pregnant women at primary facilities".

PROBLEM SUMMARY
---------------
The `tpr_precomputed.db` file is not being created after the TPR workflow
completes. This file should contain all 16 TPR combinations pre-computed for
fast switching.

The `precompute_debug.log` file (which we added for debugging) is also NOT
being created, which means the background thread function is never executing.

EXPECTED BEHAVIOR
-----------------
1. User completes TPR workflow (uploads data, selects state/facility/age group)
2. TPR is calculated for the selected combination
3. Background thread starts to pre-compute remaining 15 combinations
4. `tpr_precomputed.db` is created in the session folder
5. User can then say "Switch to pregnant women at primary facilities"
6. Tool queries the pre-computed database and regenerates files

ACTUAL BEHAVIOR
---------------
1. User completes TPR workflow - WORKS
2. TPR is calculated - WORKS
3. Background thread starts - THREAD NEVER EXECUTES (no logs, no debug file)
4. `tpr_precomputed.db` - NOT CREATED
5. User tries to switch - FAILS with "Pre-computed TPR data not found"
