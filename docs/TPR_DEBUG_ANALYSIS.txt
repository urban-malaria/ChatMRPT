================================================================================
ROOT CAUSE #1: FLAWED TPR AGGREGATION LOGIC
================================================================================

FILE: app/core/tpr_utils.py
LINES: 484-489

CURRENT CODE (WRONG):
```python
# When test_method='both':
final_tpr = max(rdt_tpr, micro_tpr)
total_tested = max(rdt_tested, micro_tested)  # ← WRONG!
total_positive = rdt_positive if rdt_tpr > micro_tpr else micro_positive
```

PROBLEM:
- Takes max of tested counts but corresponding positive count from ONE method
- Creates inconsistent ratios when RDT and Microscopy have different volumes

EXAMPLE OF BUG:
  RDT: 500 tested, 100 positive = 20% TPR
  Microscopy: 1000 tested, 120 positive = 12% TPR

  Current code calculates:
  - final_tpr = max(20%, 12%) = 20%
  - total_tested = max(500, 1000) = 1000  ← From Microscopy
  - total_positive = 100 (from RDT)
  - But 100/1000 = 10% ✗ (doesn't match claimed 20% TPR!)

CORRECT APPROACH (Volume-Weighted):
```python
# True volume-weighted TPR:
total_tested = rdt_tested + micro_tested  # OR just use one method
total_positive = rdt_positive + micro_positive
final_tpr = (total_positive / total_tested) * 100
```

ROOT CAUSE #4: LLM DOES MATH INSTEAD OF DETERMINISTIC TOOLS
================================================================================

FILE: app/data_analysis_v3/core/python_tool.py

CURRENT APPROACH:
- LLM writes arbitrary pandas code
- Executes in SecureExecutor
- LLM computes: df.groupby('WardName')['TPR'].mean()  ← Mean-of-means!

PROBLEM:
- Mean-of-means is statistically incorrect for surveillance data
- Should be: sum(positives) / sum(tested)  ← Volume-weighted

EXAMPLE OF ERROR:
  Ward A: 100 tested, 80 positive = 80% TPR
  Ward B: 1000 tested, 500 positive = 50% TPR

  Mean-of-means: (80% + 50%) / 2 = 65%
  Volume-weighted: (80 + 500) / (100 + 1000) = 52.7%  ← Correct!
